Правила сборки материала

Экспертные рекомендации для конструктора электронного курса и лонгрида: как блоки соседствуют, куда вставляются и какие ограничения держат материал понятным для ученика.

12 типов блоков Плоский список Drag-and-drop Курс + лонгрид
Материал собирается из независимых блоков в фиксированном порядке. Текст, картинки и схемы не вкладываются друг в друга — это даёт предсказуемый вид на всех устройствах, простое перетаскивание и возможность автогенерации.

0. Базовые правила архитектуры

Фиксируем один раз и не пересматриваем при отрисовке каждого блока.

Вложенность блоковНет. Только плоский списокПроще drag-and-drop, превью, тесты, AI

Текст обтекает картинкуНетНужна предсказуемая учебная вёрстка

1. Текст: внутренняя структура

В палитре одна кнопка «Текст», но в материале — четыре логических подтипа. Рекомендуем считать их отдельными блоками в списке.

Подтип Как появляется Несколько подряд Рекомендация
H2Текст + уровень заголовкаДа, но не более 1 H2 на смысловый блокНовый раздел урока — новый H2
H3Из текста / отдельная вставкаДаТолько после H2 или другого H3
АбзацКнопка «Текст» по умолчаниюДаОсновной носитель теории, 3–7 абзацев на раздел
СписокИз текста / отдельная вставкаДаПосле абзаца или H3; не два списка подряд без текста

Жёсткое правило: урок начинается с H2 (или шапка урока + первый H2 в теле).

3. Матрица соседства

Строка — что уже стоит. Столбец — что добавляем после. ✓ можно · ! не рекомендуется · ✕ запретить / предупредить

4. Рекомендуемые шаблоны порядка

Не обязательные пресеты, но золотые паттерны для авторов.

A — Теория

H2 → абзац → абзац → список → изображение → абзац → важный блок

B — Инструкция

H2 → абзац → шаги → файл → важный блок

C — Кейс / инцидент

H2 → абзац → таймлайн → абзац-вывод → важный блок

D — Справочник

H2 → абзац → таблица → абзац → глоссарий

E — Схема процесса

H2 → абзац → блок-схема → важный блок

F — Разбор объекта

H2 → абзац → марк. изображение → список → абзац
