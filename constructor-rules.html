Правила сборки материала

Экспертные рекомендации для конструктора электронного курса и лонгрида: как блоки соседствуют, куда вставляются и какие ограничения держат материал понятным для ученика.

12 типов блоков Плоский список Drag-and-drop Курс + лонгрид
Материал собирается из независимых блоков в фиксированном порядке. Текст, картинки и схемы не вкладываются друг в друга — это даёт предсказуемый вид на всех устройствах, простое перетаскивание и возможность автогенерации.

0. Базовые правила архитектуры

Фиксируем один раз и не пересматриваем при отрисовке каждого блока.

1. Текст: внутренняя структура

В палитре одна кнопка «Текст», но в материале — четыре логических подтипа. Рекомендуем считать их отдельными блоками в списке.

Подтип Как появляется Несколько подряд Рекомендация
H2Текст + уровень заголовкаДа, но не более 1 H2 на смысловый блокНовый раздел урока — новый H2
H3Из текста / отдельная вставкаДаТолько после H2 или другого H3
АбзацКнопка «Текст» по умолчаниюДаОсновной носитель теории, 3–7 абзацев на раздел
СписокИз текста / отдельная вставкаДаПосле абзаца или H3; не два списка подряд без текста

Жёсткое правило: урок начинается с H2 (или шапка урока + первый H2 в теле).

3. Можно ли ставить блоки рядом?

Эта таблица отвечает на простой вопрос: если один блок уже стоит в уроке, можно ли сразу после него поставить другой?

Как читать таблицу

  1. Найдите в левом столбце блок, который уже есть в уроке.
  2. Посмотрите на пересечение с верхней строкой — какой блок хотите добавить следом.
  3. Оцените отметку в ячейке.

Пример: после «Абзац» можно поставить «Видео» — в строке «Текст» на пересечении со столбцом «Видео» стоит ✓. А вот после «Видео» сразу ещё одно «Видео» — ✕: конструктор должен предупредить.

✓ можно без предупреждения · ! можно, но показать подсказку · ✕ лучше запретить или сильно предупредить

4. Рекомендуемые шаблоны порядка

Не обязательные пресеты, но золотые паттерны для авторов.

A — Теория

H2 → абзац → абзац → список → изображение → абзац → важный блок

B — Инструкция

H2 → абзац → шаги → файл → важный блок

C — Кейс / инцидент

H2 → абзац → таймлайн → абзац-вывод → важный блок

D — Справочник

H2 → абзац → таблица → абзац → глоссарий

E — Схема процесса

H2 → абзац → блок-схема → важный блок

F — Разбор объекта

H2 → абзац → марк. изображение → список → абзац

6. Отступы между блоками

Между элементами в уроке не нужна отдельная матрица «текст → фото → диаграмма». Достаточно одной шкалы отступов и нескольких явных исключений для заголовков и связанного текста.

Базовый принцип

Блоки урока — это вертикальный список. Контейнер задаёт единый gap между соседями. Отступ — это ритм урока, а не свойство каждой пары типов.

.lp-lesson-body { display: flex; flex-direction: column; gap: var(--space-m); }

Шкала отступов (токены)

ТокенЗначениеГде используется
--space-xs12pxВнутри блока: подпись к картинке, мелкие подписи
--space-s24pxСвязанный текст: H3 → абзац, абзац → список
--space-m40pxСтандарт между любыми двумя блоками
--space-l60pxПеред новым разделом (H2), вокруг «тяжёлых» блоков
--space-xl88pxРедко: крупное медиа, переход к тесту

Категории блоков

Между категориями — тот же --space-m, если нет исключения ниже.

КатегорияТипыВнешний отступ
ТекстH2, H3, абзац, списокm; у H2/H3 — см. исключения
Медиаизображение, видео, аудио, диаграмма, марк. изображениеm, при необходимости l сверху
Структурныешаги, таймлайн, таблица, блок-схема, глоссарийm или l — блок «тяжёлый»
Акцентважный блок, файлне меньше m сверху и снизу

Исключения (7 правил)

СитуацияОтступ
H2 — новый разделСверху l (60px), кроме первого блока в уроке; до следующего блока эффект s (24px)
H3 — подразделСверху m; до абзаца/списка подряд — s
Связанный текст подрядH3 → абзац, абзац → список: s вместо m
Медиа после текстаСтандарт m; подпись — внутри блока (xs)
Два медиа подрядНе уменьшать gap; лучше предупреждение в конструкторе
Тяжёлые и акцентные блокиСверху и снизу не меньше m
Первый / последний блокБез лишнего margin — отступ даёт padding плашки урока

Чего не делать

ПодходПочему плохо
У каждого блока свой margin-top и margin-bottomСхлопывание margin, разный вид в редакторе и превью
Матрица отступов для всех пар типовСотни значений, сложно поддерживать
Отступ зависит от ширины контента блокаНепредсказуемо для автора
Копировать Word (Enter = отступ)В блочной модели нет смысла

Спека для разработки

DEFAULT: gap между блоками = 40px (--space-m) H2: margin-top 60px (кроме первого в уроке) H3 + text/list подряд: 24px между соседями Все остальные пары: 40px Внутренние отступы — в компоненте блока, не в gap урока Редактор и превью ученика — одни и те же токены
Для заказчика: «Между элементами везде одинаковый стандартный интервал. Чуть больше — только перед новым разделом и вокруг крупных вставок. Уменьшать отступы имеет смысл только внутри связанного текста, а не для каждой пары типов блоков».
